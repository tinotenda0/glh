Technical User Guide

This is a comprehensive guide for developers, system administrators, and other technical users on the GreenField Local Hub (GLH) website. It covers setup, project structure, and key technical features.

1. Prerequisites

Before you begin, ensure you have the following installed:

  • Python
  • Django
  • Pillow (a Python imaging library)

You can install the Python libraries using pip. [1]

pip install django pillow

2. Installation & Setup

Follow these steps to get the development environment running.

  1. Create and activate a virtual environment: 
    # Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
  2. Install Dependencies: 
    pip install django pillow
  3. Apply Database Migrations: If no `db.sqlite3` file exists, run the following commands to create the database schema. [1] 
    python manage.py makemigrations
python manage.py migrate
  4. Create a Superuser (Admin): This user is required to access the Django admin and will be treated as a Seller by the seeding script. [1] 
    python manage.py createsuperuser
  5. Run the Development Server: 
    python manage.py runserver

    The site will be available at `http://127.0.0.1:8000`. [1]

3. Seeding the Database

The project includes a script to populate the database with 10 sample stores and 50 products for the admin user. [1, 3]

  1. Open the Django interactive shell: 
    python manage.py shell
  2. Open the file accounts/seed_data.py in a text editor. [1]
  3. Copy the entire contents of the file. [1]
  4. Paste the code into the Django shell and press Enter. [1]
  5. Wait for the success message and exit the shell. [1]

4. Project Structure

The project is organized into the following main directories: [1]

  • glh/: The core Django project for settings and main URL routing.
  • accounts/: The primary app handling User models, authentication, roles, stores, products, carts, orders, and business logic. [1]
  • app/: A Django app for frontend-related views like landing pages and user guides. [1]
  • media/: Directory where user-uploaded images (avatars, logos, product images) are stored. [1]
  • staticfiles/: Contains static files for the admin interface and frontend libraries like jQuery and Select2. [4, 5, 6]

5. Technical Details

User Roles & Authentication

  • The custom User model in accounts/models.py has boolean flags is_customer and is_seller to define roles. [2, 7]
  • Django's built-in `Group` model is also used ('Customer', 'Seller'). Users are added to these groups upon registration or role change. [2]
  • The @group_required decorator in accounts/views.py restricts view access based on group membership or superuser status. [2]

Custom Error Handling

  • The project includes custom templates for 404 (Not Found) and 500 (Internal Server Error) errors. [1]
  • These custom error pages are only displayed when DEBUG = False in settings.py. [1]
  • Important: Setting DEBUG = False in a development environment will break the serving of media files. This is expected Django behavior. [1]

Django Admin

The built-in Django admin interface is available at /admin. You can log in with the superuser credentials to manage users, groups, and all other database models. [1]
